# RAG‑Biography Chat: ассистент по биографиям известных личностей

# Стек
Python, LangChain, ChromaDB, Hugging Face Transformers, DuckDuckGo Search, requests, BeautifulSoup, NLP‑модель hugging‑quants/Meta-Llama-3.1-8B-Instruct-AWQ‑INT4.


# Описание проекта
В рамках проекта реализуется диалоговая система, которая отвечает на вопросы о жизни известных людей с помощью архитектуры Retrieval‑Augmented Generation (RAG). В качестве базы знаний используются первые абзацы русскоязычных статей Википедии о людях из открытого датасета. В проекте реализованы:

- построение векторной базы данных (ChromaDB) на основе первых 100 текстов;

- механизм семантического поиска по этой базе;

- функция поиска информации в интернете через DuckDuckGo (для случаев, когда данных в базе недостаточно);

- поддержка диалогового режима, позволяющая задавать уточняющие вопросы без повторного полного формулирования запроса;

- интеграция всех компонентов в единую RAG‑систему для генерации ответов.

Генеративным ядром выступает квантизованная модель Llama‑3.1‑Instruct, оптимизированная под работу с ограниченными ресурсами и способная генерировать осмысленные ответы как на русском, так и на английском языках.

# Цель проекта
Цель проекта — создать ассистента, который:

- отвечает на вопросы о биографии и жизни известных личностей, используя семантический поиск по базе знаний;

- дополняет ответы информацией из интернета при отсутствии достаточных данных в локальной базе (например, про авторов реальных курсов, которые не описаны в базе Википедии);

- поддерживает диалог, позволяя уточнять детали без повторного полного описания предыдущего вопроса;

- демонстрирует принципы построения RAG‑системы «с нуля»: от сборки векторной базы данных до интеграции внешнего поиска и генеративной модели.

Такой ассистент может служить прототипом интеллектуального помощника для быстрого поиска биографической информации и дальнейшего расширения функциональности (например, добавление UI‑интерфейса или расширение базы знаний).

# Основные компоненты системы

- Векторная база данных (ChromaDB)  
  Построена на основе первых 100 текстов из предварительно подготовленного корпуса биографических текстов о знаменитостях. Все эмбеддинги сохраняются на диск в папке `chroma_db`, что исключает повторный пересчёт при каждом запуске.

- Поиск по базе знаний  
  Приходит запрос → вычисляется вектор запроса → извлекается заданное количество наиболее релевантных фрагментов текста → используется как контекст для генеративной модели.

- Поиск информации в интернете  
  Реализована функция `internet_search`, которая по текстовому запросу и параметру `k` возвращает `k` фрагментов текста из результатов поиска DuckDuckGo с применением `requests` и `BeautifulSoup`. Интернет‑поиск включается, когда в базе знаний не найдено достаточно релевантных документов.

- Поддержка диалогового режима  
  Учитывается история диалога: пользователь может задавать уточняющие вопросы, а система связывает их с предыдущими репликами, чтобы не повторять уже известную информацию.

- RAG‑конвейер  
  Система объединяет retrieval‑часть (поиск в базе знаний + при необходимости — интернет‑поиск) с генерацией ответа на модели Llama‑3.1‑Instruct. На основе этого конвейера формируются ответы на тестовые диалоговые вопросы из файла `dialog_questions.txt`, которые сохраняются в `dialog_answers.json`.

# Используемые данные
- Предварительно подготовленный корпус биографических текстов о знаменитостях, основанный на текстах русскоязычных статей Википедии.

- Для тестирования диалога используется файл `dialog_questions.txt` с парами вопросов, на которые нужно сгенерировать ответы.

- Для кейсов, которых нет в базе (например, про современных преподавателей или неописанных в Википедии лиц), используются результаты поиска в интернете.

# Результаты и выводы
- Построена автономная векторная база данных, которая позволяет быстро извлекать релевантные фрагменты текста о знаменитостях без повторного вычисления эмбеддингов.

- Реализован гибридный механизм поиска: сначала по базе знаний, при необходимости — через интернет, что расширяет область покрытия вопросов.

- Добавлена поддержка диалога, позволяющая задавать уточняющие вопросы и получать последовательные ответы, опирающиеся на историю общения.

- В ходе тестирования показано, что система способна заменять стандартный поиск по фактам о людях, при этом сохраняя акцент на релевантность и ясность текста, а не на «подгонке» ответа под предвзятое представление.
